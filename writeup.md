I built a modular Python scraper using Playwright to extract structured tenders from a dynamic procurement portal (Option A). For this submission it targets GeM India (`bidplus.gem.gov.in`). The scraper is config-driven via `config/sites.yaml` (URLs + selectors), and is run through `python -m src.main --site gem_india --max-pages N`. It visits `/all-bids`, paginates through multiple pages, extracts key fields from each listing card, and deduplicates discovered detail links before enrichment.

From the listing page it extracts title, organization/department, posted/closing dates, and the detail link. GeM listing cards don’t consistently expose a true “price/budget”; what is reliably present is “Quantity”, so the pipeline stores that separately as `quantity` and keeps `quantity_raw` for traceability. A GeM-specific normalization step also infers `category_or_department` (Product/Service) and enriches `location` (state/city heuristics; central ministries default to New Delhi). Dates are parsed into ISO format where possible, and normalization removes label noise and collapses whitespace.

The main challenge is that on GeM the “detail page” frequently isn’t HTML at all: the card link points to a bid-document endpoint (`/showbidDocument/<id>`) that serves a PDF. The scraper treats this PDF as the detail source and performs best-effort PDF text extraction (first pages only) to capture labeled fields when they exist (e.g., “Bid Value”, “Start Date”, “End Date”, tender reference patterns). Listing and detail values are merged with a simple rule: prefer detail values when present, otherwise keep listing values.

Failures are handled without stopping the run: a robots.txt allow-check is performed up front, there are randomized delays between requests, navigation/downloads use retry/backoff, and PDF enrichment is best-effort with per-record error capture. Output is exported to both CSV and JSON under `output/`, with logs written to `output/scraper.log`.

This scraper would break if GeM changes DOM structure/selectors, pagination behavior, introduces stronger throttling/403s, or if PDF layouts/text extraction degrade. Improvements would include selector health checks, checkpoint/resume for long runs, broader PDF field extraction beyond the first pages, and small regression tests for normalization/parsing rules.
